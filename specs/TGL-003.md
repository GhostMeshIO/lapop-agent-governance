## 1. Purpose

TGL-003 defines the Temporal Governance Layer for multi-agent LLM orchestration.

It manages time, aging, rollback, checkpoints, stale branches, unresolved dependencies, oscillation, governance debt, and long-running execution lifecycle.

TGL answers the question:

> What should happen to artifacts, notes, policies, strategies, branches, and disagreements as time passes?

Multi-agent systems degrade over time when they lack temporal governance.

Common failures include:

* stale dependency notes,
* unresolved blocking branches,
* old speculative outputs contaminating mainline,
* artifacts oscillating between versions,
* repeated rollback/merge loops,
* outdated policy assumptions,
* stale strategy priors,
* abandoned branches,
* expired exceptions still influencing decisions,
* and governance debt accumulating invisibly.

TGL exists to make long-running agent systems temporally stable.

---

## 2. Design Goals

TGL-003 aims to make multi-agent systems:

* checkpointed,
* rollback-safe,
* time-aware,
* aging-aware,
* stale-branch-resistant,
* oscillation-resistant,
* governance-debt-aware,
* archival-capable,
* replay-friendly,
* and compatible with policy and strategy version pinning.

Core principle:

> A long-running agent system must know what is current, what is stale, what is reversible, and what should be archived, reopened, frozen, or rolled back.

---

## 3. Non-Goals

TGL-003 does **not**:

* execute worker tasks,
* decide factual truth,
* validate syntax,
* merge artifacts,
* override policy,
* learn strategy directly,
* compute economic value directly,
* maintain trust graphs directly,
* or modify artifacts without LAPOP/PCL approval.

TGL manages temporal state and emits temporal recommendations.

LAPOP executes approved actions.

PCL decides whether temporal actions are allowed.

SLL manages strategy ledger internals.

CHL provides collaboration health and oscillation signals.

EIL helps decide whether temporal interventions are worth it.

## 5. Core Concepts

### 5.1 Temporal State

Temporal state is the time-indexed condition of system objects:

* artifacts,
* task nodes,
* dependency notes,
* branches,
* disagreements,
* policy attestations,
* strategy ledger heads,
* checkpoints,
* exception windows,
* validation results,
* audit results.

TGL makes temporal state explicit.

---

### 5.2 Checkpoint

A checkpoint is a restorable or replayable system state.

A good checkpoint includes:

* artifact hashes,
* task state,
* policy attestation ID,
* strategy ledger head,
* dependency note state,
* validation hashes,
* tool output hashes,
* execution environment data.

---

### 5.3 Rollback Candidate

A rollback candidate is a checkpoint that may be restored.

Rollback candidates should be scored for:

* temporal consistency,
* policy compatibility,
* strategy compatibility,
* artifact integrity,
* dependency-note consistency,
* contamination status,
* replay eligibility.

---

### 5.4 Stale Branch

A stale branch is an artifact or execution branch that has aged beyond its useful window or diverged too far from current mainline.

Stale branches may be:

* archived,
* deleted if allowed,
* preserved as speculative,
* reopened,
* or converted into documentation.

---

### 5.5 Oscillation

Oscillation occurs when an artifact or decision repeatedly flips between states.

Examples:

```text
version A → version B → version A → version B
merge → rollback → merge → rollback
open note → resolved → reopened → resolved → reopened
policy exception → revoked → reinstated → revoked
```

Oscillation indicates unresolved structural conflict.

---

### 5.6 Governance Debt

Governance debt is accumulated unresolved coordination/time burden.

Examples:

* stale blocking notes,
* unresolved disagreements,
* unreviewed policy exceptions,
* old rollback candidates,
* archived branches never reviewed,
* repeated oscillations,
* expired validation results,
* outdated strategy priors.

## 7. Checkpoint Policy

### 7.1 When to Create Checkpoints

Create checkpoints:

* before merge,
* after successful validation,
* before rollback,
* before applying strategy patch,
* before policy exception activation,
* before external side effect,
* before destructive action,
* after human approval,
* at cycle boundaries.

### 7.2 Checkpoint Requirements

A checkpoint should include:

```yaml
CheckpointRequirements:
  artifact_hashes: required
  execution_state: required
  policy_attestation_id: required_if_policy_checked
  strategy_ledger_head: required_if_strategy_active
  contamination_free: required
  replay_eligible: required
  invalidation_triggers: required
```

### 7.3 Checkpoint Invalidation

A checkpoint may become invalid if:

* source artifact changes,
* policy changes incompatibly,
* strategy ledger changes incompatibly,
* tool output expires,
* validation report expires,
* contamination is later discovered,
* external side effect cannot be replayed,
* artifact dependencies changed.

---

## 8. Rollback Protocol

### 8.1 Rollback Requirements

Rollback must restore or explicitly account for:

* artifact state,
* dependency note state,
* validation/audit state,
* policy attestation,
* strategy ledger head,
* temporal lifecycle state,
* contamination status,
* branch lineage.

### 8.2 Canonical Cross-Layer Restore Interfaces

PCL must expose:

```python
restore_policy_state(attestation_id: str)
```

SLL must expose:

```python
revert_strategy_to_ledger_head(ledger_head: str)
```

TGL coordinates these calls when rollback requires policy or strategy restoration.

### 8.3 Rollback Flow

```text
RollbackCandidate
  → RollbackPlan
  → PCL policy check
  → restore artifacts
  → restore dependency notes
  → restore policy state if required
  → restore strategy state if required
  → validate restored state
  → write RollbackExecutionRecord
```

### 8.4 Rollback Safety Rules

1. Do not rollback to contaminated state unless explicitly quarantined.
2. Do not rollback policy without attestation.
3. Do not rollback strategy without ledger head.
4. Validate after rollback.
5. Preserve rollback provenance.
6. Record restored and replaced artifacts.
7. If rollback fails, freeze affected branch and escalate.

### 9.2 Aging Function

```python
def update_lifecycle_age(obj_state):
    obj_state.age_cycles += 1

    if obj_state.ttl_cycles is not None:
        if obj_state.age_cycles >= obj_state.ttl_cycles:
            obj_state.lifecycle_state = "stale"
            obj_state.stale_reason = "ttl_expired"

    return obj_state
```

## 10. Stale Branch Handling

### 10.1 Detection

A branch becomes stale when:

* TTL expires,
* mainline diverges too far,
* dependencies remain unresolved,
* policy changes,
* strategy changes,
* validation becomes old,
* owner disappears,
* evidence becomes stale.

### 10.2 Actions

| Condition              | Recommended Action   |
| ---------------------- | -------------------- |
| Low-value branch       | Archive              |
| Potential future value | Preserve speculative |
| Mainline failed        | Reopen               |
| Policy changed         | Revalidate           |
| Strategy changed       | Rebenchmark          |
| Evidence stale         | Refresh evidence     |
| Owner unavailable      | Ownership transfer   |

## 12. Oscillation Detection

### 12.1 Version Flip Detection

```python
def detect_version_flip(history, window=6):
    recent = history[-window:]
    if len(recent) < 4:
        return False

    pattern = [item.version_hash for item in recent]

    return (
        len(set(pattern)) == 2
        and pattern[0] == pattern[2]
        and pattern[1] == pattern[3]
    )
```

### 12.2 Merge/Rollback Loop

Detect when:

```text
merge → rollback → merge → rollback
```

within a configured cycle window.

### 12.3 Dependency Reopen Loop

Detect when:

```text
open → resolved → reopened → resolved → reopened
```

### 12.4 Recommended Responses

| Severity | Response                 |
| -------- | ------------------------ |
| Low      | Monitor                  |
| Medium   | Supervisor review        |
| High     | Freeze object            |
| Critical | Human review or rollback |

## 13. Deadlock Detection

TGL detects time-persistent deadlocks.

Deadlock sources:

* circular dependencies,
* blocking note loops,
* unresolved ownership transfer,
* policy conflict loops,
* validation repair loops,
* merge conflict loops.

### Dependency Cycle Example

```text
Artifact A requires change in B.
Artifact B requires change in C.
Artifact C requires change in A.
```

### Deadlock Response

TGL may recommend:

* break lowest-priority dependency,
* supervisor resolution,
* human review,
* split artifact,
* reject stale note,
* rollback,
* archive branch.

## 14. Governance Debt Scoring

Recommended formula:

```text
DebtScore =
  w1 * stale_note_ratio
+ w2 * unresolved_blocking_ratio
+ w3 * stale_branch_ratio
+ w4 * unresolved_disagreement_ratio
+ w5 * oscillation_rate
+ w6 * deadlock_rate
+ w7 * expired_exception_ratio
+ w8 * overdue_review_ratio
```

Example:

```python
def compute_governance_debt(report):
    return min(1.0,
        0.15 * report.stale_note_count
        + 0.20 * report.unresolved_blocking_note_count
        + 0.10 * report.stale_branch_count
        + 0.15 * report.unresolved_disagreement_count
        + 0.15 * report.oscillation_count
        + 0.15 * report.deadlock_count
        + 0.05 * report.expired_policy_exception_count
        + 0.05 * report.overdue_review_count
    )
```

In a real implementation, counts should be normalized against active object counts.

## 23. Contamination Rules

TGL must track contamination status temporally.

A clean artifact can become suspect later if:

* upstream evidence is invalidated,
* a source is discovered fabricated,
* a branch is reopened from contaminated context,
* a rollback candidate points to contaminated state,
* a validation report is found invalid.

### Contaminated Rollback Rule

Do not rollback to contaminated state unless:

* explicitly quarantined,
* action is advisory or diagnostic,
* human/supervisor approval exists,
* and the restored state is not promoted to mainline.

## 24. Failure Modes

| Failure Mode                 | Description                               | Default Response                    |
| ---------------------------- | ----------------------------------------- | ----------------------------------- |
| Stale branch pollution       | Old branch contaminates mainline          | Require revalidation                |
| Rollback to bad state        | Restores contaminated or invalid artifact | Block and escalate                  |
| Policy mismatch rollback     | Artifact restored under wrong policy      | Restore policy attestation or block |
| Strategy mismatch rollback   | Artifact restored under wrong strategy    | Restore strategy head or block      |
| Infinite reopen loop         | Archived branch repeatedly reopened       | Freeze and review                   |
| Oscillation blindness        | Version flip not detected                 | Add oscillation monitor             |
| Over-archival                | Useful branch archived too early          | Preserve speculative                |
| Governance debt invisibility | Debt grows unreported                     | Emit debt report                    |
| Checkpoint bloat             | Too many checkpoints                      | Archive/prune by policy             |
| Expired exception linger     | Old exception still affects decisions     | Force expiration                    |

## 25. Abuse and Misconfiguration Risks

### 25.1 Rollback Abuse

Users or agents may rollback to avoid valid validation failures.

Mitigation:

* policy check,
* rollback reason,
* validation after rollback,
* rollback execution record.

### 25.2 Stale Branch Resurrection

Old branches may be revived without revalidation.

Mitigation:

* revalidation required,
* policy/strategy compatibility check,
* contamination check.

### 25.3 Oscillation Suppression

System may freeze valid iteration as oscillation.

Mitigation:

* severity thresholds,
* supervisor review,
* allow intentional exploration labels.

### 25.4 Checkpoint Sprawl

Too many checkpoints may overwhelm storage.

Mitigation:

* TTL policy,
* pruning,
* archival,
* compression,
* checkpoint importance score.

### 25.5 Governance Debt Ignored

Reports may be generated but never acted on.

Mitigation:

* debt thresholds,
* automatic intervention recommendations,
* constitutional review triggers.

def temporal_cycle_update(context, execution_records):
    lifecycle_states = load_temporal_lifecycle_states(context)

    for obj_state in lifecycle_states:
        update_lifecycle_age(obj_state)

        if is_stale(obj_state):
            apply_stale_action(obj_state)

    checkpoints = collect_recent_checkpoints(context)
    rollback_candidates = [
        score_rollback_candidate(cp)
        for cp in checkpoints
        if cp.replay_eligible and cp.contamination_free
    ]

    stale_branches = detect_stale_branches(context)
    oscillations = detect_oscillations(context)
    deadlocks = detect_deadlocks(context)

    debt_report = compute_governance_debt_report(
        context=context,
        stale_branches=stale_branches,
        oscillations=oscillations,
        deadlocks=deadlocks,
        lifecycle_states=lifecycle_states,
    )

    status = TemporalStatusReport(
        context=context,
        active_objects=count_state(lifecycle_states, "active"),
        stale_objects=count_state(lifecycle_states, "stale"),
        archived_objects=count_state(lifecycle_states, "archived"),
        frozen_objects=count_state(lifecycle_states, "frozen"),
        rollback_candidates=len(rollback_candidates),
        oscillations_detected=len(oscillations),
        deadlocks_detected=len(deadlocks),
        governance_debt_score=debt_report.debt_score,
        recommended_interventions=debt_report.recommended_actions,
    )

    return status

## 28. Rollback Pseudocode

```python
def execute_rollback(plan):
    policy_result = pcl.is_action_allowed(
        context=current_context(),
        action=ActionDescriptor(
            action_type="rollback",
            side_effect_class="write_internal",
            metadata={"rollback_plan_id": plan.rollback_plan_id},
        ),
    )

    if not policy_result.allowed:
        return RollbackExecutionRecord(
            rollback_plan_id=plan.rollback_plan_id,
            rollback_successful=False,
            failure_reason="policy_denied",
        )

    candidate = load_rollback_candidate(plan.rollback_candidate_id)

    if not candidate.contamination_free:
        return fail_rollback(plan, "candidate_contaminated")

    restored_artifacts = restore_artifacts(candidate.artifact_refs)

    restored_policy = None
    if plan.restore_policy_state and candidate.policy_attestation_id:
        restored_policy = pcl.restore_policy_state(candidate.policy_attestation_id)

    restored_strategy = None
    if plan.restore_strategy_state and candidate.strategy_ledger_head:
        restored_strategy = sll.revert_strategy_to_ledger_head(
            candidate.strategy_ledger_head
        )

    validation_reports = []
    if plan.validation_required_after:
        validation_reports = validate_restored_artifacts(restored_artifacts)

    success = all(v.schema_valid and not v.contamination_detected for v in validation_reports)

    return RollbackExecutionRecord(
        rollback_plan_id=plan.rollback_plan_id,
        policy_result_id=policy_result.query_id,
        restored_artifacts=restored_artifacts,
        restored_policy_attestation_id=candidate.policy_attestation_id,
        restored_strategy_ledger_head=candidate.strategy_ledger_head,
        validation_reports_after=validation_reports,
        rollback_successful=success,
        failure_reason=None if success else "post_rollback_validation_failed",
    )
```

## 32. Summary

TGL-003 is the timekeeper of the stack.

It makes long-running multi-agent systems aware of:

* age,
* freshness,
* rollback,
* checkpoints,
* stale branches,
* unresolved notes,
* oscillation,
* deadlock,
* policy state,
* strategy state,
* and governance debt.

Without TGL, agent systems can appear functional while slowly accumulating stale, contradictory, irreversible state.

TGL prevents time from becoming an invisible source of corruption.
