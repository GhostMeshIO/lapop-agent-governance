## 1. Purpose

SLL-003 defines the contamination-safe strategy learning layer for multi-agent LLM orchestration.

It learns **how the system should work better** from clean execution outcomes.

It does not learn task truth.

It does not update model weights.

It does not silently rewrite artifacts.

It does not use contaminated runs.

SLL-003 answers the question:

> Given past clean executions, which routing, repair, validation, merge, escalation, or stop strategies should be preferred in future runs?

SLL exists because long-running agent systems should improve over time, but naive learning is dangerous. If a system learns from hallucinated, contaminated, boundary-violating, or failed runs, it will poison its future behavior.

SLL’s core principle:

> Learn only from clean runs, update only explicit strategy tables, and make every learned update reversible.

---

## 2. Design Goals

SLL-003 aims to make strategy learning:

* contamination-safe,
* explicit,
* reversible,
* benchmark-gated,
* auditable,
* scope-bounded,
* rollback-compatible,
* and separate from artifact truth.

The layer should improve operational behavior such as:

* which agent class to route to,
* when to ask for review,
* which repair pattern works,
* when to preserve disagreement,
* which validation threshold catches errors,
* when another retrieval/tool call is worth trying,
* and when to stop.

---

## 3. Non-Goals

SLL-003 does **not**:

* update model weights,
* invent factual memory,
* rewrite artifacts directly,
* override policy,
* bypass validation,
* decide human safety policy,
* perform temporal rollback,
* compute economic value by itself,
* or treat generated output as ground truth.

SLL consumes clean execution records and emits strategy update patches.

It does not own policy authority, artifact authority, or truth authority.

---

## 4. Core Responsibilities

| Responsibility            | Description                                              |
| ------------------------- | -------------------------------------------------------- |
| Outcome Collection        | Read execution records and extract outcomes.             |
| Quarantine Gate           | Exclude contaminated, invalid, or policy-violating runs. |
| Feature Extraction        | Convert clean runs into strategy features.               |
| Routing Prior Learning    | Improve agent-class routing choices.                     |
| Repair Pattern Mining     | Learn which repair strategies work for which failures.   |
| Validation Tuning         | Suggest validation threshold adjustments.                |
| Escalation Learning       | Learn when human/supervisor review is useful.            |
| Merge Regret Tracking     | Track when merge choices were later reversed.            |
| Stop/Continue Learning    | Learn when more computation stops adding value.          |
| Strategy Patch Generation | Emit explicit, reversible strategy updates.              |
| Benchmark Gate            | Test patches before they become active.                  |
| Strategy Ledger           | Append strategy updates to reversible ledger.            |
| Rollback Support          | Restore prior strategy head when TGL requests rollback.  |

## 6. Clean-Run Requirement

SLL only learns from runs where:

```yaml id="tk9oex"
contamination_free: true
validation_passed: true
policy_denied: false
boundary_violation_unresolved: false
execution_record_complete: true
```

SLL must reject or quarantine runs with:

* contamination detected,
* missing execution record,
* unresolved boundary violation,
* unresolved policy conflict,
* hallucinated source,
* failed validation,
* incomplete provenance,
* unknown memory eligibility,
* benchmark failure.

## 9. Quarantine Gate

### 9.1 Required Checks

```python id="zq0oxq"
def is_learning_eligible(record):
    if not record.contamination_free:
        return False, "contamination_detected"

    if not all(v.syntax_valid and v.schema_valid for v in record.validation_reports):
        return False, "validation_failed"

    if any(v.contamination_detected for v in record.validation_reports):
        return False, "validation_contamination"

    if any(a.contamination_detected for a in record.audit_reports):
        return False, "audit_contamination"

    if unresolved_boundary_violation(record):
        return False, "boundary_violation"

    if unresolved_policy_conflict(record):
        return False, "policy_conflict"

    if not record.reproducibility_profile:
        return False, "missing_provenance"

    return True, "eligible"
```

### 9.2 Quarantine Outcomes

If not eligible, SLL writes `LearningQuarantineRecord`.

Quarantined runs may be used for:

* negative examples,
* failure taxonomies,
* benchmark construction,
* manual review.

They may not be used to improve active strategy tables without review.

## 10. Outcome Normalization

SLL converts heterogeneous execution records into `StrategyOutcomeRecord`.

Normalization should remove task-specific content while preserving operational features.

Example:

```text id="0p3wty"
Raw:
  Worker repaired qnvm_0.7.1.py CSV bug after one traceback.

Normalized:
  task_type = code_repair
  artifact_type = file
  failure_type = runtime_exception
  repair_count = 1
  validation_passed = true
  outcome_quality_score = 0.92
```

SLL does not store the factual content of the file as truth.

It stores the strategy pattern.

## 14. Benchmark Gate

No strategy patch becomes active until it passes benchmark.

### 14.1 Required Benchmarks

A patch should be tested against:

* clean success cases,
* contaminated cases,
* boundary violation cases,
* policy conflict cases,
* high-cost cases,
* human reversal cases,
* merge disagreement cases,
* regression examples.

### 14.2 Gate Criteria

A patch fails if it:

* increases contamination rate,
* increases unresolved boundary violations,
* increases human reversal rate significantly,
* decreases quality beyond tolerance,
* causes policy bypass,
* breaks rollback,
* over-escalates to humans,
* increases cost without sufficient quality gain.

### 14.3 Benchmark Result

Patch must produce `BenchmarkGateResult`.

Only if `passed: true` may the strategy ledger commit it.

## 15. Strategy Memory Ledger

The ledger head is referenced by:

## 23. Contamination Rules

SLL must never treat generated claims as source evidence.

Examples of forbidden learning:

```text id="seadnp"
A critic hallucinated that function X was broken.
Repair succeeded accidentally.
SLL learns "function X is broken often."
```

Allowed learning:

```text id="9ac7ii"
Runtime traceback showed CSV field union error.
Patch fixed it.
Validation passed.
SLL learns repair strategy for CSV field union errors.
```

Key distinction:

> SLL learns operational patterns from validated outcomes, not factual truth from generated text.

## 26. Stop/Continue Learning

SLL can help identify when further work stops helping.

Signals:

* repeated repair failure,
* no quality improvement after N attempts,
* cost rising faster than quality,
* audit findings unchanged,
* human reversal after over-iteration,
* validation already passed with high confidence.

SLL emits hints.

## 27. Failure Modes

| Failure Mode                   | Description                                   | Default Response                     |
| ------------------------------ | --------------------------------------------- | ------------------------------------ |
| Learning from contaminated run | Bad data enters strategy                      | Quarantine and rollback patch        |
| Overfitting strategy           | Patch improves benchmark but harms generality | Broaden benchmark, reduce confidence |
| Reward hacking                 | Strategy optimizes easy metric                | Add counter-metrics                  |
| Boundary weakening             | SLL recommends unsafe permission              | PCL denial                           |
| Human over-escalation          | Too many human reviews                        | EIL cost review                      |
| Tool overuse                   | Strategy routes too many tool calls           | EIL and benchmark penalty            |
| Merge bias                     | Always accepts one agent class                | CHL/SLL audit                        |
| Benchmark leakage              | Patch tuned to benchmark only                 | holdout benchmark                    |
| Non-reversible patch           | Strategy cannot rollback                      | reject patch                         |
| Hidden artifact mutation       | SLL changes content                           | policy violation                     |

## 28. Abuse and Misconfiguration Risks

### 28.1 Poisoned Learning

A contaminated run is incorrectly marked clean.

Mitigation:

* strict quarantine gate,
* independent audit,
* contamination flag in record,
* benchmark regression.

### 28.2 Strategy Capture

One agent class accumulates excessive routing preference.

Mitigation:

* diversity constraints,
* confidence decay,
* CHL trust cross-check.

### 28.3 Hidden Policy Override

Strategy patch effectively bypasses policy by routing around restrictions.

Mitigation:

* PCL policy check before patch application,
* policy audit of proposed patch.

### 28.4 Regression Through Goodharting

Patch improves one metric while degrading others.

Mitigation:

* multi-metric benchmark gate,
* cost/quality/contamination/boundary metrics.

### 28.5 Human Preference Overfitting

System overfits to one reviewer.

Mitigation:

* role-level aggregation,
* privacy constraints,
* explicit policy approval for personalization.

## 30. Reference Pseudocode

```python id="f1106v"
def process_execution_record_for_learning(record):
    eligible, reason = is_learning_eligible(record)

    if not eligible:
        quarantine = LearningQuarantineRecord(
            execution_id=record.execution_id,
            reason=reason,
            details="Execution record failed learning eligibility.",
            reusable_for_negative_training=True,
        )
        store_quarantine_record(quarantine)
        return None

    outcome = normalize_execution_record(record)
    features = extract_strategy_features(outcome)

    candidate_patch = mine_strategy_patch(features, outcome)

    if candidate_patch is None:
        return None

    policy_result = pcl.is_action_allowed(
        context=record.context,
        action=ActionDescriptor(
            action_type="update_strategy",
            side_effect_class="write_internal",
            metadata={"patch_id": candidate_patch.patch_id},
        ),
    )

    if not policy_result.allowed:
        return reject_patch(candidate_patch, policy_result.reason)

    gate_result = run_benchmark_gate(candidate_patch)

    if not gate_result.passed:
        return reject_patch(candidate_patch, "benchmark_failed")

    ledger_entry = apply_strategy_patch(candidate_patch, gate_result)

    return ledger_entry
```

## 34. Summary

SLL-003 is the learning layer, but it learns carefully.

It does not learn from everything.

It does not trust generated text by default.

It does not mutate artifacts.

It does not override policy.

It learns operational strategy from clean, validated, auditable execution records.

Its purpose is to make the system better over time without letting bad runs poison the future.

SLL is not memory.

SLL is disciplined adaptation.
